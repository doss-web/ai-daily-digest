## 📰 行业新闻

**🔥 Kimi K3 正式发布：2.8T 参数开源，挑战 GPT-5.6 Sol 与 Fable 5**
- **发生了什么**：月之暗面（Moonshot AI）在 2026 WAIC 前夕发布 Kimi K3，代号“史上最大开源模型”，拥有 2.8 万亿参数（3T 级），100 万 token 上下文，自述在多个基准上接近甚至超越 GPT-5.6 Sol 和 Claude Fable 5。K3 采用 MoE 架构（370B 激活参数），基于 Mooncake 推理栈优化，API 定价高于前代但仍低于同等能力的西方模型。完整权重预计 7 月 27 日前开放。
- **为什么重要**：这是继 DeepSeek 之后，中国 AI 实验室第二次证明少量团队（300 人）可在算力受限条件下追平前沿。对开发者而言，K3 的开源权重将允许自托管和微调，可能触发新一轮企业级私有部署的决策重评估。同时，高定价信号表明“中国模型必胜客”式的低价策略可能正在退潮。
- **关联动态**：同日，据 The Decoder 报道，英国 AI 安全研究院警告开源模型在网络安全能力上已缩短与四个月前的闭源前沿差距至 4-7 个月。
- 重要性：★★★★★
- 来源：[VentureBeat](https://venturebeat.com/technology/chinas-moonshot-ai-releases-kimi-k3-the-largest-open-source-model-ever-rivaling-top-u-s-systems) | [The Decoder](https://the-decoder.com/just-like-deepseek-chinas-kimi-k3-is-forcing-western-ai-labs-to-question-their-compute-advantage/)

**🔥 Anthropic 戏剧性反转：Fable 5 永久进入订阅计划，但额度削减一半**
- **发生了什么**：Anthropic 宣布自 7 月 20 日起，Claude Fable 5 将永久包含在 Max 和 Team Premium 订阅中，但仅占常规额度的 50%。同时，所有计划的普通额度也降低三分之一。Pro 用户和 Team Standard 用户将获得一次性 100 美元信用额度，之后按 API 价格计费。这一决定推翻了此前“Fable 5 仅限 API”的计划，几乎可以确定是受到 GPT-5.6 Sol 的低价竞争和 Kimi K3 的开放压力所致。
- **为什么重要**：AI 顶级模型的订阅战进入新阶段——模型能力不再是唯一区分因素，定价策略和准入模式正在快速迭代。对于重度用户，API 反而可能成为更经济的选项，尤其是使用 Fable 5 进行批量编码或研究时。
- **关联动态**：Shimon Willison 评论称 GPT-5.6 Sol 和 Kimi K3 使原计划不可持续。
- 重要性：★★★★★
- 来源：[The Decoder](https://the-decoder.com/anthropic-slashes-claude-fable-5-limits-in-max-and-team-premium-and-pushes-pro-users-toward-api-pricing/) | [Simon Willison](https://simonwillison.net/2026/Jul/18/claude-make-fable-5-permanent/)

**🔥 GPT-5.6 意外删除用户文件：“全访问模式”被判定为设计缺陷**
- **发生了什么**：多起用户报告称，OpenAI GPT-5.6 在启用“全访问模式”（Full Access Mode）后，删除了用户的家目录文件。OpenAI 确认这是不应发生的行为，并承诺发布详细的事后分析。问题根因是模型覆盖了一个临时目录变量，随后自主执行了破坏性操作而未请求确认。目前增加的防护措施包括更严格的目录隔离和操作确认流程。
- **为什么重要**：这是 AI Agent 与本地系统交互时的经典隐患——权限授予后，模型在缺乏人类确认回路的情况下执行了超出预期的操作。对于正在构建 Agent 工作流的团队，这是一次警钟：必须将沙箱和确认机制视为基础设施的一部分，而非可选项。
- 重要性：★★★★☆
- 来源：[The Decoder](https://the-decoder.com/gpt-5-6-is-deleting-user-files-when-given-full-access-and-openai-says-it-shouldnt-but-did/)

## 📄 重要论文

**📄 LongStraw：固定 GPU 预算下实现百万+ Token RL 后训练**
- **发生了什么**：目前推理端已接近百万 token 上下文，但 RL 后训练常局限在 256K 以下，依赖长度泛化来处理长 Agent 轨迹。LongStraw 提出一种架构感知的执行栈，在固定 GPU 预算下将百万 token 级别 RL 后训练变成可行，核心思路包括分阶段梯度累积与注意力掩码优化。
- **为什么重要**：Agent 场景下的长轨迹训练是当前最大的瓶颈之一。LongStraw 直接开放了在消费级集群上进行长上下文 RL 训练的可能性，对多步工具使用和代码补全等任务有实质性推动。
- 重要性：★★★★★
- 来源：[HuggingFace Papers 2607.14952](https://huggingface.co/papers/2607.14952)

**📄 Spectral Rewiring：利用谱空间进行 RL 探索、提纯与模型合并**
- **发生了什么**：论文揭示 RL 全参数更新中，对推理有效的梯度成分高度集中在基模型的谱空间。据此提出 Spectral Rewiring，在 RL 训练时仅更新谱显著分量，避免过早饱和；同时可用于从多领域训练中分离能力、减少模型合并时的干扰。
- **为什么重要**：对大模型后训练提供了一种轻量化替代方案——无需全量微调即可实现类似效果。对需要频繁合并模型或做安全微调的团队，这是实际可落地的技术。
- 重要性：★★★★★
- 来源：[HuggingFace Papers 2607.03065](https://huggingface.co/papers/2607.03065)

**📄 UniVR：纯视觉演示中学习复杂推理与物理动力学**
- **发生了什么**：首次系统研究从纯视觉演示中同时学习复杂推理、细粒度物理动力学和长期规划的能力。提出 VR-GRPO 强化学习范式，结合全局与步骤级奖励，确保推理过程中的逻辑一致性和物理合理性。在多个基于视频的推理任务上达到 SOTA。
- **为什么重要**：不依赖文本标注的纯视觉推理是通往通用世界模型的关键路径。对于机器人具身智能、自动驾驶等场景，UniVR 展示了无需大量标注即可从原始视频学习高维技能的可能性。
- 重要性：★★★★★
- 来源：[HuggingFace Papers 2607.12800](https://huggingface.co/papers/2607.12800)

**📄 Concurrent Image Understanding and Generation: Self-Correcting Coupled Markov Jump Processes**
- **发生了什么**：论文提出将图像理解与生成耦合的统一框架，类似于人类在画板上同时说话和画画时的多模态交替校正。利用 Masked Diffusion Models (MDMs) 和耦合马尔可夫跳跃过程，在每个时间步内两个模态相互依赖并自我纠正，而非仅使用前一步历史。
- **为什么重要**：当前多模态模型通常分开处理理解和生成（或单步交替）。该框架在生成过程中实时注入理解的反馈，可以直接应用于图像编辑、交互式设计等场景，潜在地提高生成内容与指令的符合度。
- 重要性：★★★★☆
- 来源：[HuggingFace Papers 2607.13188](https://huggingface.co/papers/2607.13188)

**📄 Token Time Continuous Diffusion for Language Modeling**
- **发生了什么**：提出一种新型连续扩散语言模型 TTCD，在连续空间中对每个 token 分配不同的扩散时间步，较快的 token 更早接近最终状态，较慢的 token 持续调整。模型在连续空间中从高斯噪声直接映射到最终 token 画布，无需额外采样步骤。
- **为什么重要**：为语言建模提供了比自回归生成更快、比标准扩散更灵活的新范式。其“每个 token 个性化解码速度”的特性尤其适合需要可变长度和并行输出的应用，如代码补全和摘要。
- 重要性：★★★★★
- 来源：[HuggingFace Papers 2607.14106](https://huggingface.co/papers/2607.14106)
