## 📄 重要论文（选 5 条）

**📄 SpectraReward：预训练多模态大模型作为零样本奖励模型，用于文本到图像生成的强化学习**
- **发生了什么**: 提出一种无需训练（training-free）的奖励函数 SpectraReward，通过计算图像条件提示的对数似然来衡量生成图像与原始提示的“可回复性”。无需额外的奖励模型、人工标注或分解问题，仅需一次向前传播。
- **为什么重要**: 当前图像生成 RL 依赖独立奖励模型或人工反馈，成本高、泛化弱。SpectraReward 让任意预训练 MLLM 瞬间成为奖励模型，极大降低 RL 微调门槛。对想用 RL 优化扩散模型的团队是直接可用工具。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11886)

**📄 Know Before Fix：QA 驱动的仓库知识获取，提升代码 Agent 修复能力**
- **发生了什么**: 针对 LLM Agent 在处理代码仓库问题时因语境不足导致的事实错误，提出先识别 Agent 的知识缺口，再通过结构化 QA（问题和答案对）引导仓库探索，而非盲目地预读文件。在 SWE-bench 等 benchmark 上提升明显。
- **为什么重要**: 这是第一个将“知识缺口感知”引入代码修复 Agent 的方法，解决了当前 Agent 要么扫描整个代码库（浪费 tokens）、要么遗漏关键信息的痛点。开发者可将此思路集成到自己的 Agent 脚手架中。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11111)

**📄 多智能体 LLM 无法有效探索彼此：短视和极化交互模式导致协作不良**
- **发生了什么**: 形式化“多智能体探索问题”，建模为部分可观测随机博弈（POSG）。实验表明，当前主流 LLM Agent（GPT-4, Claude 等）在多智能体交互中表现出短视和极化模式，无法有效探索彼此的策略空间，导致后悔值偏高、协作次优。
- **为什么重要**: 随着多 Agent 系统（如编程团队、科研协作）日益流行，这一发现指出当前 LLM Agent 存在系统性的探索缺陷，需新的交互协议或训练方法。对构建 Multi-Agent 框架的团队是直接警示。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11250)

**📄 PUST：代理引导更新信号传输 —— 模块化 LLM 后训练新范式**
- **发生了什么**: 提出 Proxy-guided Update Signal Transfer（PUST），将策略探索与分布对齐解耦。用小模型做探索（rollout 更便宜），将优化信号传输到大模型，支持异步生成、复用和跨模型转移，训练效率大幅提升。
- **为什么重要**: 当前的 RLVR（可验证奖励强化学习）对每个新模型都要重新 rollout，成本极高。PUST 让后训练可以不重复探索，直接重用信号，对频繁迭代模型的团队（如每周微调）是架构级优化。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11505)

**📄 Do AI Agents Know When a Task Is Simple? 面向复杂度感知的推理与执行**
- **发生了什么**: 发现当前 LLM Agent 倾向于“最大上下文优先”策略 —— 即使是单行编辑也会遍历整个代码库。论文提出任务执行范围估计（task-aware execution-scope estimation），让 Agent 先判断任务难度，再决定需要哪些信息和最短路径。
- **为什么重要**: 这是对当前 Agent 过度消耗 tokens 问题的直接诊断和解决方案。估算难度并动态调整搜索范围，可在不损失准确性的情况下大幅降低成本。对 Agent 的 Token 预算管理和成本效率具有直接工程价值。
- 重要性：★★★★★
- 来源：[arXiv](https://arxiv.org/abs/2607.13034v1)
