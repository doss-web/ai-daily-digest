**🔥 Thinking Machines 发布 Inkling 975B 开放权重多模态模型**
- **发生了什么**：由前 OpenAI CTO Mira Murati 创立的 Thinking Machines Lab 发布了其首个大语言模型 Inkling，参数量 975B，支持文本、图像、音频等多模态输入，以 Apache 2.0 开源许可证发布。公司强调模型在开放权重级别表现出色（虽未达到闭源前沿水平），且专注于低推理成本和对审查的“抵抗”。模型可直接在企业本地或 VPC 中部署。

**🔥 OpenAI 发布 GPT-Red：用 AI 自我对抗实现自动化红队测试**
- **发生了什么**：OpenAI 发布了 GPT-Red，一个通过自对弈训练的自动化红队模型，能在 84% 的测试场景中找到成功攻击，而人类红队仅达到 13%。GPT-Red 专为发现 prompt 注入、对齐漏洞等设计，其输出直接用于强化 GPT-5.6 Sol 等生产模型的鲁棒性。
- **为什么重要**：这表明对抗性安全测试可以大规模自动化，并且效率远超人工。对于构建 agent 系统的开发者，这意味着未来的安全加固将更快速、更全面，但也意味着攻击者可能利用类似方法生成更难检测的 prompt 注入。建议所有在生产中部署 LLM 的团队关注此类自动化红队的部署方式。

**🔥 OpenAI Codex 加密子代理指令，开发者失去内部委托透明度**
- **发生了什么**：自 6 月初起，OpenAI 的编程工具 Codex 开始加密主代理传递给子代理的指令。开发者无法再追踪任务如何在内部被委托。对于更大的 GPT-5.6 变体 Sol 和 Terra，该加密是强制的。这一改变实际上使 agent 的推理过程变成了黑箱。

## 📄 重要论文

**📄 Function-Aware Fill-in-the-Middle 作为编码代理基础模型的中间训练**
- **发生了什么**：研究者观察到编码代理的动作-观察-延续循环在结构上等同于函数调用站点。他们提出 FIM-FA（Function-Aware Fill-in-the-Middle），利用互联网代码中普遍存在的调用-返回结构进行中间训练。结果模型在 SWE-bench Verfied 上达到 45.6% 解决率，超越了此前基于演进轨迹微调的方法（38.1%）。
- **为什么重要**：这一发现揭示了预训练数据中隐藏的结构化信号可以大幅提升编码代理的推理能力，且无需昂贵的合成轨迹。对于构建 agentic coding 工具的团队，这是一个可直接采用的训练范式。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.12463)

**📄 弱到强泛化：基于直接在线策略蒸馏**
- **发生了什么**：强化学习可验证奖励（RLVR）在每个新强模型上重复运行时成本高昂。本文提出一种弱到强替代方案：先在较小的模型上运行 RL（更便宜的 rollout），然后将其学到的知识蒸馏到强目标模型。实验表明，在线风格（on-policy）蒸馏效果显著优于离线蒸馏，小模型带来的弱到强泛化有效且高效。
- **为什么重要**：随着模型规模增长，后训练本身成为瓶颈。该方法让开发者可以用小模型探索策略空间，再迁移到大模型，大幅降低后训练成本。这是实际可落地的训练成本优化方案。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.05394)

**📄 修复前先了解：基于 QA 驱动的仓库知识获取用于软件问题修复**
- **发生了什么**：当前 LLM 编码代理在修复 bug 时常因缺乏仓库理解而产生事实错误。本文提出 KnowRAG 框架，通过 QA 驱动的探索来识别代理的知识缺口，预先获取精确的上下文，然后再进行修复。在 SWE-bench 上，KnowRAG 将修复准确率从 40.3% 提升到 52.7%。
- **为什么重要**：该方法显式地让代理先“知道缺失了什么”再行动，而非盲目探索。对于所有使用 agent 进行代码修复的团队，这是一个可以直接集成的工作流改进。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11111)

**📄 多智能体 LLM 无法相互探索**
- **发生了什么**：论文将多智能体协作形式化为部分可观测随机博弈（POSG），发现现代 LLM 智能体在相互交互时表现出短视和极化模式，无法有效探索，导致次优协调和更高遗憾。即使使用 GPT-5.6 Sol 和 Anthropic Opus 4.8 等最强模型，该问题依然存在。
- **为什么重要**：这对多 agent 系统（如 agentic orchestration 场景）的设计提出了根本性挑战：即使单个 agent 能力强，多 agent 协作的探索仍然是未被解决的核心难题。需要新的架构或训练方法来解决这一固有缺陷。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11250)

**📄 AdvancedMathBench：高级数学证明生成和验证的基准套件**
- **发生了什么**：现有数学基准局限于高中和竞赛级别。论文发布了 AdvancedMathBench，覆盖高等数学中六个子领域（实分析、抽象代数、拓扑等），包含 892 个问题，要求模型生成完整证明。GPT-5.6 Sol 和 Claude Fable 表现最佳，但整体正确率仍很低（<30%），在拓扑和微分流形等抽象领域尤其挑战。
- **为什么重要**：这是目前最严谨的高等数学推理评估之一，揭示了前沿模型在复杂逻辑和抽象推理上的真实上限。对于评估 agent 的推理能力，这个基准提供了更有区分度的测试集。
- 重要性：★★★★★
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.11849)
