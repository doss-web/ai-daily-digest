## 📄 重要论文

**📄 Stanford 提出 TRACE：将 Agent 重复失败转化为合成 RL 环境的训练系统**
- **发生了什么**：斯坦福研究者开发了 TRACE，该框架从 Agent 自身轨迹中诊断能力缺失，为每个缺失合成一个可验证的训练环境，训练对应的 LoRA 适配器，并在推理时路由 Token 到对应专家。在 τ²-Bench 上提升+15.3 分，SWE-bench Verified 达到 73.2% Pass@1。
- **为什么重要**：提供了一种自动化持续改进 Agent 能力的工程化方法，特别适合需要反复迭代的 AI 开发团队。可据此构建内部 Agent 技能积累的流水线。
- 重要性：★★★★★
- 来源：[MarkTechPost](https://www.marktechpost.com/2026/07/13/stanford-researchers-introduce-trace/)

**📄 KronQ：通过 Kronecker 分解 Hessian 矩阵实现 LLM 量化**
- **发生了什么**：KronQ 提出在量化目标中引入梯度协方差，而非仅依赖激活统计。通过 Kronecker 因式分解近似二阶信息，在 perplexity 和下游任务上显著优于 GPTQ 等现有方法。
- **为什么重要**：压缩是部署大型模型的关键瓶颈。KronQ 提供了更紧凑的量化方案，对资源受限的设备推理或大规模部署可减少显存占用和延迟，值得嵌入工具链。
- 重要性：★★★★☆
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.07964)

**📄 Requential Coding：用自生成训练数据突破模型压缩极限**
- **发生了什么**：论文提出 prequential coding 框架，将压缩视为序列预测任务，利用模型对自身训练数据生成概率码。实验表明，该方法在模型大小和下游性能之间实现了更好的帕累托前沿，优于参数级压缩方法（如量化、剪枝）。
- **为什么重要**：改变了压缩的视角——不在于参数量，而在于数据表示的信息量。对 AI 研究员而言，可能导向全新的模型存储与传输协议；对开发者意味着未来模型可能更小更好。
- 重要性：★★★★★
- 来源：[arXiv](https://arxiv.org/abs/2607.11883)

**📄 Video Generation Models are General-Purpose Vision Learners（GenCeption）**
- **发生了什么**：论文主张大规模文本到视频生成可作为计算机视觉的通用预训练范式，提出 GenCeption 框架，在视频生成预训练之上微调即可统一各类视觉任务（分类、检测、分割、跟踪等），表现优于专门模型。
- **为什么重要**：呼应了“视频生成是视觉世界的 next-token prediction”的论点，可能重新定义视觉基座模型训练路线。对做多模态的团队，应关注该方向对下游任务的影响。
- 重要性：★★★★☆
- 来源：[HuggingFace Papers](https://huggingface.co/papers/2607.09024)

**📄 Self-Guided Test-Time Training（Self-TTT）：无需额外数据的实时长上下文优化**
- **发生了什么**：提出在测试阶段将长上下文视为训练实例，利用模型自身生成伪标签进行参数更新，无需额外标注数据。在多个长上下文基准（如 LongBench）上显著提升性能，且延迟可控。
- **为什么重要**：解决模型看似支持长上下文但实际使用不佳的问题。对使用长文档、代码库、对话历史的应用，可尝试集成 Self-TTT 作为运行时优化层，提升准确性。
- 重要性：★★★★☆
